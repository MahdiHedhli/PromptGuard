## Completed today

- **Random token IDs (DEC-012).** Format `[CATEGORY_<16-hex>]`, generated by `secrets.token_hex(8)` for 64 bits of entropy. The reverse-path regex is `\[[A-Z][A-Z_]*_[a-f0-9]{16,}\]`. Sequential IDs were a prompt-injection vector per threat-model A7; the random IDs collapse the attack to ~2^63 expected guesses. Existing tests that hardcoded `[EMAIL_001]` updated to match by regex.
- **ConversationTokenMap with TTL + LRU eviction (DEC-013).** In-memory dict keyed by `conversation_id`. Either condition is sufficient to evict: last access older than `PROMPTGUARD_TOKEN_MAP_TTL_S` (default 3600) or the map holds more than `PROMPTGUARD_TOKEN_MAP_MAX_CONVERSATIONS` (default 100). Idempotent within a conversation: re-issuing the same `(category, original)` returns the same token. Cross-conversation isolation: distinct conversations always get distinct random suffixes. 14 dedicated unit tests in `tests/unit/test_token_map.py` cover format, idempotency, isolation, TTL eviction (with a monkey-patched `time.monotonic`), LRU eviction, and concurrent issue+restore under contention.
- **Reverse path (non-streaming).** `TokenMap.restore(cid, text)` runs a single regex sub against the conversation's own map. Tokens not in the map pass through unchanged ("never invent reverse mappings", per the brief). Walker `_restore_strings_in_object` traverses dict / list / typed-object responses (LiteLLM's typed `ModelResponse`-like classes) and substitutes string fields in place.
- **Reverse path (streaming).** Two primitives + one coalescing helper (DEC-014):
  - `StreamingRestorer` (text-stream, 256-char buffer with partial-token detection): chunk-boundary independent, validated by feeding test inputs split at every byte position.
  - `SSEStreamRestorer` (event-level, walks `delta.text` and `choices[].delta.content` per event, restores in JSON-aware fashion).
  - `restore_sse_blob(token_map, cid, bytes)`: the path the live iterator hook actually uses. Parses every event, concatenates text deltas, restores in the concatenation, re-emits. Required because LiteLLM hands the iterator hook a single buffered `bytes` chunk for Anthropic native streaming, and tokens can span multiple delta events when the upstream chunks at small boundaries.
- **LiteLLM post-call hooks.** `async_post_call_success_hook` (non-streaming) and `async_post_call_streaming_iterator_hook` (streaming, both string/bytes/typed-chunk) are bound onto the core `PromptGuardHook`. `async_post_call_streaming_hook` and `async_post_call_streaming_deployment_hook` are also bound for completeness; LiteLLM may exercise either depending on the route. Conversation ID and request ID are persisted into `data["metadata"]` by the pre-call so the post-call can recover them.
- **Mock Anthropic upstream.** New compose service `mock-anthropic` (`docker/mock_anthropic/`) under the `mock` profile. Anthropic-shaped `/v1/messages` that echoes any tokens it sees in the prompt back into a small reply, supports both non-streaming and streaming responses, exposes `/_test/last_received` for the integration tests to assert what the upstream actually saw. Built and used today; staying in the repo as a fixture for Days 4+.

## Code / artifact highlights (blog post fodder)

- **The "tokens straddle SSE events" gotcha.** Worth at least a paragraph in the post. The mock chunks at 4 chars to mimic worst-case Anthropic; a single token like `[INTERNAL_IP_a3f9c1d2e4b56789]` then spans 8 separate `content_block_delta` events. Naively scanning each event for the token finds zero matches because the raw event bytes contain JSON envelope characters between every fragment. Three approaches considered, one shipped (DEC-014).
- **The threat-model A6/A7 rewrite.** Now reads like a layered-defense argument. Random IDs collapse the prompt-injection attack to a 2^63 brute force; per-conversation scoping enforces correctness rather than relying on probability; pure string substitution means the LLM cannot select a key. Three layers, each independently sufficient to break the attack.
- **Latency snapshot for the post.**
  - In-process Day-1 baseline (regex only): avg 0.071ms / p95 0.075ms.
  - In-process Day-2 (pipeline + action engine + JSON-safe rewrite): avg 0.154ms / p95 0.271ms.
  - Day-3 end-to-end through proxy, non-streaming clean prompt: p95 15.6ms / p99 17.2ms.
  - Day-3 end-to-end through proxy, TOKENIZE round-trip (PII inbound, restore outbound): p95 22.6ms / p99 102.5ms (one outlier in 30 samples).
  - Day-3 streaming TTFB: p95 24.5ms; total p95 25.7ms. PromptGuard's contribution to this is in the iterator hook, on the order of single-digit ms.
- **Per-conversation map shape.** 1-hour TTL plus 100-conversation LRU bounds the on-disk forensic surface to a one-hour rolling window per conversation. Memory upper bound at worst-case load is ~25MB; bounds exist for retention reasons not memory reasons.
