## Code / artifact highlights (blog post fodder)

- **Adapter framework as a CISO talking point.** The repo now has a clear story: detectors decide what is sensitive, policy sources decide what to do about it, both are adapter interfaces, and you swap any of them with one env-var change. The blog post section 9 ("Adapter framework + DLP integration") gets a worked example: change `PROMPTGUARD_POLICY_SOURCE=local_yaml` to `=purview_dlp` and the same proxy now sources its policy from a Purview SIT export. The wire shape of the export is shipped as a fixture so consumers see exactly what the integration consumes.
- **Equivalence test is a real reassurance.** `test_purview_and_yaml_agree_on_subset_of_rules` constructs both a YAML policy and a Purview-export policy with the same intent and proves they make identical decisions across the confidence axis. This is the kind of test a security buyer asks for when comparing competing DLP integrations.
- **The "fail loud" rule is now structural.** Both adapter ABCs document it as a binding rule. PurviewDLPPolicy raising `PolicySchemaError("informationTypes[2] missing or invalid 'promptguardCategory': 'not_a_real_category'")` looks like the LocalYAMLPolicy error format because they share the raise path. Consistent operator experience across sources.
- **Two synthetic fixtures shipped:** [`sample-sit-export.json`](../tests/fixtures/purview/sample-sit-export.json) and [`sample-rules-response.txt`](../tests/fixtures/icap/sample-rules-response.txt). The Purview fixture has a `_v1_1_todo` field documenting the live-fetch work explicitly; the ICAP fixture has a comment describing the vendor-translation strategy. Both can lift directly into the blog as illustrative artifacts.
- **Real-key capture as upstream-error-passthrough proof.** `Credit balance is too low` came back unchanged from Anthropic. PromptGuard's error envelope owns the BLOCK path and only the BLOCK path; everything else passes through. Worth a short footnote in the post about non-interference with upstream error semantics.
