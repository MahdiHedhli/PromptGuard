- **AuditWriter + AuditEvent** at [`src/promptguard/audit/writer.py`](../src/promptguard/audit/writer.py). JSONL append-only, lock-serialized writes, opens at startup. Required fields: timestamp (ISO 8601 UTC), conversation_id, request_id, rule, detector, category, span (offset/length), would_have_been_action, pipeline_version, policy_hash, confidence. The offending text is NOT in the dataclass; cannot be added without changing the schema.

- **Engine `audit_only` mode.** When `policy.audit_only: true`, the action engine runs in dry-run: detections fire, decisions are computed, the text is forwarded unchanged, one audit event is appended per non-ALLOW detection. When `audit_only: false`, no events are written even when a writer is wired (so flipping audit_only via hot-reload (DEC-016) changes behavior immediately).

## Code / artifact highlights (blog post fodder)

- **The fuzz invariant is the right kind of test for a security tool.** 200 iterations of random PII-like strings asserted absent from the audit log file content under any code path. Stdlib only, runs in 0.1 seconds, has caught nothing yet (correctly), and tells security reviewers "we have programmatic evidence of the no-text promise." Worth a paragraph in the blog post about test posture: "we treat the audit log as a sensitive datastore and test it like one."
- **`pipeline_version + policy_hash`** in every audit event is a small idea that scales well. Operators reviewing audit logs across releases can pin their analysis to specific code/policy revisions without having to query a separate registry. SHA-256 is overkill for a 5KB YAML file, but it is the universal log-format expectation. Worth a sidebar.
- **LLM judge tolerance posture.** A strict implementation would crash the pipeline on a malformed JSON response. Ours returns zero detections + warning. The blog can use this as a worked example of "design every dependency to be tolerable when it fails": the deterministic stages stay correct without the judge, the judge is a recall booster, not a precision floor.
- **Real-key round-trip is in the can.** The curl-based artifact at `docs/blog-assets/03-real-anthropic-roundtrip.txt` shows the upstream got the random token, the user got the original IP back, and Anthropic billed 28 input + 20 output tokens. The asset is reproducible; readers can replay it themselves with the same prompt and see the same shape.
